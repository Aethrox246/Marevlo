01 — Why Convolutions?

The Problem with Flat MLPs on Images

A 224×224 RGB image has 224 × 224 × 3 = 150,528 pixels. If you flatten it and pass it to a linear layer with 512 hidden units, you already have 77 million parameters — just for the first layer. Worse, the MLP treats each pixel independently, ignoring the fact that nearby pixels are related.

Three CNN Inductive Biases
Local Connectivity · Weight Sharing · Translation Invariance
Local connectivity — each neuron only sees a small patch of the input, not the whole image  ·  Weight sharing — the same filter is applied at every position (dramatically fewer parameters)  ·  Translation invariance — a cat in the top-left is the same as a cat in the bottom-right
02 — nn.Conv2d Deep Dive

Understanding the Parameters

Output Spatial Size Formula
H_out = ⌊(H_in + 2·padding − kernel_size) / stride + 1⌋
H_in = input height  ·  padding = pixels added around border  ·  kernel_size = filter size  ·  stride = step size between filter applications
Python
import torch
import torch.nn as nn

# Conv2d(in_channels, out_channels, kernel_size, stride, padding)
conv = nn.Conv2d(3, 64, kernel_size=3, stride=1, padding=1)

# Input: (batch, channels, height, width)
x = torch.randn(8, 3, 32, 32)   # 8 RGB images 32×32
out = conv(x)
print("Input:", x.shape)    # (8, 3, 32, 32)
print("Output:", out.shape) # (8, 64, 32, 32) -- same spatial, more channels

# Parameter count: out_ch * (in_ch * k * k + bias)
params = sum(p.numel() for p in conv.parameters())
print(f"Params: {params:,}")  # 64*(3*3*3 + 1) = 1,792

# Pooling reduces spatial size
pool = nn.MaxPool2d(kernel_size=2, stride=2)
print("After pool:", pool(out).shape)  # (8, 64, 16, 16)
Output
Input:  torch.Size([8, 3, 32, 32])
Output: torch.Size([8, 64, 32, 32])
Params: 1,792
After pool: torch.Size([8, 64, 16, 16])
03 — Building a CNN from Scratch

A Modern CNN Architecture

Python
class ConvBlock(nn.Module):
    """Conv → BatchNorm → ReLU — the standard building block."""
    def __init__(self, in_ch, out_ch, kernel_size=3, stride=1):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(in_ch, out_ch, kernel_size, stride, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True),
        )
    def forward(self, x): return self.block(x)


class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            ConvBlock(3,  32),
            ConvBlock(32, 32),
            nn.MaxPool2d(2),         # /2
            ConvBlock(32, 64),
            ConvBlock(64, 64),
            nn.MaxPool2d(2),         # /4
            ConvBlock(64, 128),
            nn.AdaptiveAvgPool2d(1), # Global avg pool → (B, 128, 1, 1)
        )
        self.head = nn.Sequential(
            nn.Flatten(),
            nn.Dropout(0.3),
            nn.Linear(128, num_classes),
        )

    def forward(self, x):
        return self.head(self.features(x))


model = SimpleCNN(num_classes=10)
x = torch.randn(16, 3, 32, 32)   # CIFAR-10 batch
print("Output shape:", model(x).shape)
total = sum(p.numel() for p in model.parameters())
print(f"Total params: {total:,}")
Output
Output shape: torch.Size([16, 10])
Total params: 148,810

Transfer Learning with torchvision

Python
import torchvision.models as models

# Load pre-trained ResNet50 (ImageNet weights)
backbone = models.resnet50(weights='IMAGENET1K_V1')

# Freeze all layers
for p in backbone.parameters():
    p.requires_grad = False

# Replace final layer for your task (10 classes)
in_features = backbone.fc.in_features    # 2048
backbone.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(in_features, 10)
)

# Only the new head is trainable
trainable = sum(p.numel() for p in backbone.parameters() if p.requires_grad)
print(f"Trainable params: {trainable:,}")  # Only 20,490
Output
Trainable params: 20,490

Transfer Learning Strategy: (1) Freeze all layers, train only the head for 5–10 epochs. (2) Unfreeze the last few blocks and fine-tune with a very small LR (1e-5 to 1e-4). This "gradual unfreezing" prevents catastrophic forgetting of ImageNet features.
