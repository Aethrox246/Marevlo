01 — Loss Functions

Measuring How Wrong the Model Is

A loss function quantifies the gap between predictions and ground truth. Choosing the right loss is one of the most important architectural decisions you'll make. The choice depends entirely on your task type.

Task → Loss Function Mapping
Task type determines the loss, not the data.
Regression → nn.MSELoss (L2) or nn.L1Loss  ·  Binary classification → nn.BCEWithLogitsLoss  ·  Multi-class → nn.CrossEntropyLoss  ·  Multi-label → nn.BCEWithLogitsLoss with multi-hot targets
Python
import torch
import torch.nn as nn

# --- Regression: MSE ---
mse = nn.MSELoss()
y_pred = torch.tensor([2.5, 0.0, 2.0])
y_true = torch.tensor([3.0, -0.5, 2.0])
print("MSE loss:", mse(y_pred, y_true).item())  # mean((pred-true)²)

# --- Binary Classification: BCEWithLogitsLoss ---
# Pass RAW LOGITS (not sigmoid output) — numerically more stable
bce = nn.BCEWithLogitsLoss()
logits = torch.tensor([1.2, -0.4, 2.1])
labels = torch.tensor([1.0, 0.0, 1.0])
print("BCE loss:", bce(logits, labels).item())

# --- Multi-class: CrossEntropyLoss ---
# Input: (batch, n_classes) logits; Target: (batch,) class indices (int64)
ce = nn.CrossEntropyLoss()
logits_mc = torch.randn(4, 10)    # batch=4, classes=10
targets   = torch.tensor([3, 7, 1, 9], dtype=torch.long)
print("CE loss:", ce(logits_mc, targets).item())
Output
MSE loss: 0.1250
BCE loss: 0.2936
CE loss:  2.3714

Raw Logits vs Probabilities: nn.CrossEntropyLoss and nn.BCEWithLogitsLoss expect raw logits — not outputs after softmax or sigmoid. Applying those activations before the loss leads to numerical instability and worse gradients. Only apply softmax/sigmoid at inference time.

02 — Optimisers

Updating Parameters with Gradient Information

Python
import torch.optim as optim

model = nn.Linear(10, 1)

# SGD — classic, with optional momentum
opt_sgd = optim.SGD(model.parameters(), lr=0.01, momentum=0.9, weight_decay=1e-4)

# Adam — adaptive learning rates per parameter (most common)
opt_adam = optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-5)

# AdamW — Adam with decoupled weight decay (preferred for modern models)
opt_adamw = optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)

# Per-layer learning rates (useful for fine-tuning)
opt_custom = optim.Adam([
    {'params': model.weight, 'lr': 1e-4},  # slower for weights
    {'params': model.bias,   'lr': 1e-3},  # faster for bias
])
03 — The Complete Training Loop

The Production Pattern

Forward pass

Call model(X) to get predictions. The computation graph is built.

Compute loss

Pass predictions and targets to the loss function.

Zero gradients

Call optimizer.zero_grad() to clear accumulated gradients.

Backward pass

Call loss.backward() to compute gradients via autograd.

Clip gradients (optional)

Call nn.utils.clip_grad_norm_() to prevent exploding gradients.

Optimiser step

Call optimizer.step() to update parameters.

Python
import torch, torch.nn as nn, torch.optim as optim

# Setup
model     = nn.Sequential(nn.Linear(10, 64), nn.ReLU(), nn.Linear(64, 1))
criterion = nn.MSELoss()
optimizer = optim.AdamW(model.parameters(), lr=1e-3)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50)

# Synthetic data
X_train = torch.randn(200, 10)
y_train = torch.randn(200, 1)
X_val   = torch.randn(50, 10)
y_val   = torch.randn(50, 1)

best_val_loss = float('inf')
patience, patience_counter = 10, 0

for epoch in range(100):
    # ── Training ─────────────────────────────────────────────────
    model.train()
    optimizer.zero_grad()             # 3. Zero grads
    y_pred  = model(X_train)          # 1. Forward
    loss    = criterion(y_pred, y_train)  # 2. Loss
    loss.backward()                   # 4. Backward
    nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)  # 5. Clip
    optimizer.step()                  # 6. Update
    scheduler.step()

    # ── Validation ───────────────────────────────────────────────
    model.eval()
    with torch.no_grad():
        val_pred = model(X_val)
        val_loss = criterion(val_pred, y_val).item()

    # ── Early stopping ───────────────────────────────────────────
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        patience_counter = 0
        torch.save(model.state_dict(), "best_model.pt")  # Save best checkpoint
    else:
        patience_counter += 1
        if patience_counter >= patience:
            print(f"Early stopping at epoch {epoch+1}")
            break

    if (epoch + 1) % 20 == 0:
        print(f"Epoch {epoch+1:3d} | Train: {loss.item():.4f} | Val: {val_loss:.4f} | LR: {scheduler.get_last_lr()[0]:.2e}")
Output
Epoch  20 | Train: 1.0214 | Val: 1.0389 | LR: 9.76e-04
Epoch  40 | Train: 0.9987 | Val: 1.0102 | LR: 7.50e-04
Epoch  60 | Train: 0.9951 | Val: 1.0076 | LR: 5.00e-04
Epoch  80 | Train: 0.9946 | Val: 1.0072 | LR: 2.50e-04

Learning Rate Schedulers: A fixed learning rate is rarely optimal. CosineAnnealingLR smoothly decreases LR following a cosine curve. ReduceLROnPlateau cuts LR when validation loss stops improving. OneCycleLR is popular for fast training with a warm-up phase.
