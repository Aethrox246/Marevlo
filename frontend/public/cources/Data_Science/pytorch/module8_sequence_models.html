01 — The Sequence Modelling Problem

Order Matters

Tabular data has no inherent order. Images have spatial structure. Sequential data — text, audio, time series, genomics — has temporal order: the meaning of each element depends on what came before. Standard MLPs treat every input independently and cannot capture this.

RNN Core Equation
h_t = tanh(W_h · h_{t-1} + W_x · x_t + b)
h_t = hidden state at time t (the "memory")  ·  h_{t-1} = previous hidden state  ·  x_t = input at time t  ·  The same weights W_h and W_x are used at every time step (weight sharing across time)

Vanishing Gradients in RNNs: When sequences are long (100+ steps), gradients must travel back through every time step. They either explode or vanish, making it impossible to learn long-range dependencies. This is the core motivation for LSTM and GRU.

import torch
import torch.nn as nn

# Input format: (seq_len, batch_size, input_size)  [default]
# Or: (batch_size, seq_len, input_size) if batch_first=True

batch_size = 32
seq_len    = 50
input_size = 10
hidden_size = 64
num_layers  = 2

x = torch.randn(batch_size, seq_len, input_size)

# LSTM
lstm = nn.LSTM(input_size, hidden_size, num_layers,
               batch_first=True, dropout=0.2, bidirectional=False)
out, (h_n, c_n) = lstm(x)
print("LSTM output:", out.shape)   # (32, 50, 64) — all hidden states
print("LSTM h_n:", h_n.shape)     # (2, 32, 64)  — final hidden (2 layers)
print("LSTM c_n:", c_n.shape)     # (2, 32, 64)  — final cell state

# GRU (simpler — no cell state)
gru = nn.GRU(input_size, hidden_size, num_layers, batch_first=True)
out_g, h_n_g = gru(x)
print("GRU output:", out_g.shape)  # (32, 50, 64)

03 — Sequence Classifier

Text / Time-Series Classification

Python
class SequenceClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_size, num_classes, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(
            embed_dim, hidden_size, num_layers,
            batch_first=True, dropout=0.3, bidirectional=True
        )
        self.classifier = nn.Sequential(
            nn.Dropout(0.3),
            nn.Linear(hidden_size * 2, num_classes),  # *2 for bidirectional
        )

    def forward(self, x):
        # x: (batch, seq_len) integer token IDs
        emb = self.embedding(x)             # (B, T, embed_dim)
        out, (h_n, _) = self.lstm(emb)      # h_n: (2*num_layers, B, H)
        # Take the final hidden state from both directions
        h_fwd = h_n[-2]   # Last forward layer
        h_bwd = h_n[-1]   # Last backward layer
        h = torch.cat([h_fwd, h_bwd], dim=-1)  # (B, 2*H)
        return self.classifier(h)

model = SequenceClassifier(vocab_size=10000, embed_dim=128,
                             hidden_size=256, num_classes=3)
tokens = torch.randint(1, 10000, (16, 80))   # batch=16, seq_len=80
logits = model(tokens)
print("Output:", logits.shape)   # (16, 3)
Output
Output: torch.Size([16, 3])

When to Use Transformers Instead: For sequences longer than ~200 tokens or when you need to model complex long-range dependencies, use transformers (Module 12 / attention mechanisms). LSTMs are still excellent for real-time inference on resource-constrained devices due to their sequential nature.
