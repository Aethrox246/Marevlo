01 — Regularisation Techniques

Fighting Overfitting

L2 Weight Decay
Use: Always, especially with Adam
Adds λ·||W||² to loss, penalising large weights. Set via weight_decay in optimiser.
Dropout
Use: MLPs, after dense layers
Randomly zeros neurons (p=0.1–0.5). Forces the network to learn redundant, robust representations.
Label Smoothing
Use: Classification, large datasets
Replace hard (0,1) targets with (ε/K, 1-ε). Prevents overconfidence. Use label_smoothing=0.1 in CrossEntropyLoss.
Data Augmentation
Use: When data is limited
Artificially increases training set via random transforms. Most effective regularisation for images.
Early Stopping
Use: Always as a safety net
Stop training when val loss stops improving. Save best checkpoint with torch.save(model.state_dict()).
Batch Normalisation
Use: Deep networks (5+ layers)
Normalises layer inputs across the batch. Stabilises training, allows higher LRs. Standard in modern CNNs.
02 — Weight Initialisation

Starting in the Right Place

Poor initialisation can cause vanishing or exploding activations before training even begins. PyTorch layers have good defaults, but knowing how to customise is important for non-standard architectures.

Python
import torch
import torch.nn as nn

def init_weights(module):
    if isinstance(module, nn.Linear):
        nn.init.kaiming_normal_(module.weight, mode='fan_out', nonlinearity='relu')
        if module.bias is not None:
            nn.init.zeros_(module.bias)
    elif isinstance(module, nn.Conv2d):
        nn.init.kaiming_normal_(module.weight, mode='fan_out', nonlinearity='relu')
    elif isinstance(module, (nn.BatchNorm2d, nn.LayerNorm)):
        nn.init.ones_(module.weight)
        nn.init.zeros_(module.bias)

model = nn.Sequential(nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, 10))
model.apply(init_weights)   # Apply recursively to all submodules
print("Weights initialised successfully")
Output
Weights initialised successfully
Initialisation Guide
ReLU → Kaiming Normal  ·  Tanh/Sigmoid → Xavier  ·  Transformer → Xavier Uniform
Kaiming (He) — accounts for ReLU's dead-neuron effect. Default for most modern networks.  ·  Xavier (Glorot) — maintains activation variance for symmetric activations (tanh, sigmoid).
03 — Mixed Precision Training

Train Faster, Use Less Memory

Mixed precision training uses float16 (or bfloat16) for forward/backward passes and float32 only for the optimiser update. This typically gives 2–3× speedup on modern GPUs with minimal accuracy loss.

Python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()           # Prevents fp16 underflow

for X, y in train_loader:
    optimizer.zero_grad()

    with autocast():               # Forward in fp16
        logits = model(X)
        loss   = criterion(logits, y)

    scaler.scale(loss).backward()  # Scale loss before backward
    scaler.unscale_(optimizer)    # Unscale before grad clipping
    nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)        # Update only if grads are finite
    scaler.update()              # Adjust scale factor

When to Use Mixed Precision: Always when training on NVIDIA GPUs (Ampere or newer — RTX 30xx/A100/H100). For older GPUs (pre-Volta), skip it. For Apple Silicon, use torch.mps instead of CUDA but mixed precision is less impactful.
