01 — The nn.Module Base Class

Why Does nn.Module Exist?

In Module 2, you managed w and b as raw tensors. For a real network with millions of parameters across dozens of layers, this becomes unmanageable. nn.Module solves the organisation and bookkeeping problem.

What nn.Module Gives You
Parameters · Submodules · State Dict · Device Management
.parameters() — automatically collects all learnable tensors  ·  .state_dict() — save / load all weights  ·  .to(device) — move the entire model to GPU in one call  ·  .train() / .eval() — toggle training vs inference mode
Python
import torch
import torch.nn as nn

# Minimal custom module
class LinearRegression(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features)

    def forward(self, x):
        return self.linear(x)

model = LinearRegression(1, 1)
print(model)
print("\nParameters:")
for name, param in model.named_parameters():
    print(f"  {name}: {param.shape} | requires_grad={param.requires_grad}")
Output
LinearRegression(
  (linear): Linear(in_features=1, out_features=1, bias=True)
)

Parameters:
  linear.weight: torch.Size([1, 1]) | requires_grad=True
  linear.bias:   torch.Size([1])    | requires_grad=True

The Two Rules of nn.Module: (1) Always call super().__init__() first. (2) Define all learnable sub-layers in __init__, not in forward(). If you assign a layer in forward(), it won't be tracked by .parameters().

02 — nn.Sequential vs Custom Modules

Two Ways to Stack Layers

Python
# ── Method 1: nn.Sequential ──────────────────────────────────────
# Best for simple, linear pipelines where data flows straight through
model_seq = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(256, 128),
    nn.ReLU(),
    nn.Linear(128, 10),
)

# ── Method 2: Custom nn.Module ────────────────────────────────────
# Better when you need skip connections, multiple inputs/outputs,
# conditional logic, or reusable blocks
class MLP(nn.Module):
    def __init__(self, in_dim, hidden_dim, out_dim, dropout=0.3):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden_dim)
        self.fc2     = nn.Linear(hidden_dim, hidden_dim)
        self.fc3     = nn.Linear(hidden_dim, out_dim)
        self.relu    = nn.ReLU()
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        x = self.dropout(self.relu(self.fc1(x)))
        x = self.dropout(self.relu(self.fc2(x)))
        return self.fc3(x)  # No activation on final layer (raw logits)

model = MLP(in_dim=784, hidden_dim=256, out_dim=10)
x = torch.randn(32, 784)   # Batch of 32 MNIST-like images
out = model(x)
print("Output shape:", out.shape)  # (32, 10) — logits for 10 classes
Output
Output shape: torch.Size([32, 10])
03 — Essential Layers Reference

The Layers You'll Use Every Day

nn.Linear

Fully connected layer. Computes xWT + b. Core of all MLPs and transformer feed-forward blocks.

nn.ReLU / GELU / SiLU

Activation functions. ReLU: max(0,x). GELU: smoother, used in Transformers. SiLU: used in modern models like Mistral.

nn.Dropout

Randomly zeros p fraction of neurons during training. Disabled automatically in .eval() mode.

nn.BatchNorm1d/2d

Normalises activations per batch. Stabilises training. Behaves differently in train vs eval mode.

nn.Embedding

Lookup table mapping integer indices to dense vectors. Foundation of NLP models and recommender systems.

nn.Conv2d

2D convolution for image data. Parameters: in/out channels, kernel size, stride, padding.

Python
# Inspecting model internals
model = MLP(784, 256, 10)

# Total parameter count
total = sum(p.numel() for p in model.parameters())
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"Total params:     {total:,}")
print(f"Trainable params: {trainable:,}")

# Save and load weights
torch.save(model.state_dict(), "model.pt")

new_model = MLP(784, 256, 10)
new_model.load_state_dict(torch.load("model.pt"))

# Move to GPU (if available)
device = "cuda" if torch.cuda.is_available() else "cpu"
model = model.to(device)

# Freeze a layer (stop gradients flowing through it)
for param in model.fc1.parameters():
    param.requires_grad = False

frozen_count = sum(p.numel() for p in model.parameters() if not p.requires_grad)
print(f"Frozen params: {frozen_count:,}")
Output
Total params:     268,554
Trainable params: 268,554
Frozen params:    200,960

train() vs eval() Mode

Always set the correct mode! model.train() enables Dropout and makes BatchNorm use batch statistics. model.eval() disables Dropout and makes BatchNorm use running statistics. Forgetting to call model.eval() during inference leads to randomly different predictions every run.

Python
# Standard pattern
model.train()   # Before training loop
for batch in train_loader:
    ...

model.eval()    # Before validation / inference
with torch.no_grad():
    for batch in val_loader:
        ...
