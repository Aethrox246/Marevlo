01 — Common Error Patterns

The Bugs You'll See 90% of the Time

RuntimeError: Expected all tensors to be on the same device

Model is on GPU but input is on CPU (or vice versa). Fix: X = X.to(device) and y = y.to(device) at the start of every batch. Or move data to the device in your Dataset's __getitem__.

RuntimeError: Expected input batch_size (N) to match target batch_size (M)

Shape mismatch between predictions and labels. Most commonly: model outputs (B, 1) but loss expects (B,). Fix: output.squeeze(-1) before the loss.

Loss is NaN from the first iteration

Usually caused by: (1) log(0) in numerical computations — add epsilon. (2) Passing softmax output to CrossEntropyLoss (double-softmax). (3) Exploding gradients — add gradient clipping. (4) Bad data with NaN/Inf values.

Loss doesn't decrease at all

Common causes: (1) Forgot to call loss.backward(). (2) Learning rate too high or too low. (3) Targets are wrong dtype (float instead of long). (4) Model is in eval mode during training. (5) Gradients are zero — forgot requires_grad=True.

CUDA out of memory

Accumulating tensors in a Python list inside the training loop keeps their computation graphs alive. Fix: call .item() to convert loss to a Python scalar, or .detach().cpu() for tensors you're storing for logging.

02 — Debugging Tools

Your Debugging Arsenal

# ── 1. Shape debugging ─────────────────────────────────────────────
def shape_hook(name):
    def hook(module, inp, out):
        print(f"{name}: {inp[0].shape} → {out.shape}")
    return hook

model = nn.Sequential(nn.Linear(10, 64), nn.ReLU(), nn.Linear(64, 5))
for name, layer in model.named_children():
    layer.register_forward_hook(shape_hook(name))

dummy = torch.randn(4, 10)
out = model(dummy)

# ── 2. Gradient norm monitoring ─────────────────────────────────────
def grad_norm(model):
    total = 0.0
    for p in model.parameters():
        if p.grad is not None:
            total += p.grad.data.norm(2).item() ** 2
    return total ** 0.5

# ── 3. Check for NaN ────────────────────────────────────────────────
def check_nan(tensor, name=""):
    if torch.isnan(tensor).any():
        print(f"NaN detected in {name}!")
        return True
    if torch.isinf(tensor).any():
        print(f"Inf detected in {name}!")
        return True
    return False

The Sanity Check Protocol: Before any full training run, (1) overfit a single batch — if your model can't memorise 1 batch, something is fundamentally wrong. (2) Check gradient flow with grad_norm(). (3) Verify outputs are in the expected range. (4) Log the first 10 val predictions manually.

03 — Model Profiling

Finding Speed Bottlenecks

Python
from torch.profiler import profile, record_function, ProfilerActivity
import time

# Simple timing
def benchmark(model, input_tensor, n=100):
    model.eval()
    with torch.no_grad():
        # Warmup
        for _ in range(10):
            _ = model(input_tensor)

        start = time.perf_counter()
        for _ in range(n):
            _ = model(input_tensor)
        end = time.perf_counter()

    ms_per_batch = (end - start) / n * 1000
    print(f"Avg latency: {ms_per_batch:.2f} ms/batch")

model = nn.Linear(512, 10)
x = torch.randn(64, 512)
benchmark(model, x)
