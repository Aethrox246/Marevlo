01 — Why Accuracy Is Not Enough

The Metric Selection Problem

Accuracy tells you what fraction of predictions were correct. But with imbalanced classes (e.g. 95% negative, 5% positive), a model that always predicts "negative" gets 95% accuracy while being completely useless. You need metrics aligned with your actual business goal.

Precision
TP / (TP + FP)
Of all positive predictions, how many were correct? Use when false positives are costly (spam filters).
Recall
TP / (TP + FN)
Of all actual positives, how many did we catch? Use when false negatives are costly (cancer detection).
F1 Score
2 · (P · R) / (P + R)
Harmonic mean of Precision and Recall. Balanced metric for imbalanced datasets.
ROC-AUC
Area under ROC curve
Threshold-independent. Probability that the model ranks a random positive above a random negative.
02 — Computing Metrics with PyTorch

From Logits to Evaluation

Python
import torch
import torch.nn as nn

def evaluate_classification(model, loader, device):
    model.eval()
    all_preds, all_labels, all_probs = [], [], []

    with torch.no_grad():
        for X, y in loader:
            X, y = X.to(device), y.to(device)
            logits = model(X)                       # raw logits
            probs  = torch.softmax(logits, dim=1)  # probabilities
            preds  = logits.argmax(dim=1)          # predicted class

            all_preds.append(preds.cpu())
            all_labels.append(y.cpu())
            all_probs.append(probs.cpu())

    preds  = torch.cat(all_preds)    # shape: (N,)
    labels = torch.cat(all_labels)   # shape: (N,)
    probs  = torch.cat(all_probs)    # shape: (N, C)

    # Accuracy
    accuracy = (preds == labels).float().mean().item()

    # Confusion matrix (binary case)
    tp = ((preds == 1) & (labels == 1)).sum().item()
    fp = ((preds == 1) & (labels == 0)).sum().item()
    fn = ((preds == 0) & (labels == 1)).sum().item()
    tn = ((preds == 0) & (labels == 0)).sum().item()

    precision = tp / (tp + fp + 1e-8)
    recall    = tp / (tp + fn + 1e-8)
    f1        = 2 * precision * recall / (precision + recall + 1e-8)

    return {
        'accuracy':  accuracy,
        'precision': precision,
        'recall':    recall,
        'f1':        f1,
        'confusion': [[tn, fp], [fn, tp]]
    }

Regression Metrics: For regression tasks, use nn.MSELoss (training) but report RMSE (root MSE — same units as target) and MAE (mean absolute error — more robust to outliers) in evaluation. R² score tells you the proportion of variance explained.

03 — Overfitting Diagnosis

Reading the Train–Val Gap

Diagnosis Patterns
Train loss ↓↓ | Val loss → (stuck) = Overfitting
Both losses high → Underfitting (model too simple or insufficient training)  ·  Train ↓, Val ↑ → Overfitting (model memorising training data)  ·  Both losses ↓ together → Good generalisation  ·  Val loss spiky → Try larger batch size or learning rate warmup

Test Set Contamination: Never look at your test set during model development. If you use test performance to make decisions (tuning hyperparameters, selecting architectures), you are effectively training on the test set. Keep it completely hidden until your final evaluation.
