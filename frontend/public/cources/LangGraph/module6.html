02 · Self-Improvement

Reflection Pattern — Self-Improving Agents

The source compares reflection to a writer who drafts, rereads critically, and then revises. The agent generates a response, a critic evaluates it, and if it is not good enough, the generator tries again using the feedback.

This pattern is called out as especially valuable for essays, code, and analysis, where the gap between a first draft and an improved revision can be dramatic.

def generate(state):
    """Generate or revise the response."""
    response = llm.invoke([
        SystemMessage("Write the best response you can. If you received feedback, improve based on it."),
        *state["messages"]
    ])
    return {"messages": [response], "iteration": state.get("iteration", 0) + 1}

def critique(state):
    """Critically evaluate the response."""
    review = llm.invoke([
        SystemMessage(
            "Evaluate this response critically. "
            "If it is excellent and complete, say APPROVED. "
            "Otherwise, give specific, actionable feedback for improvement."
        ),
        *state["messages"]
    ])
    return {"messages": [review]}

def should_continue(state) -> str:
    if state["iteration"] >= 3: return "done"  # Max 3 attempts
    if "APPROVED" in state["messages"][-1].content.upper(): return "done"
    return "revise"  # Loop back!

graph = StateGraph(ReflectionState)
graph.add_node("generate", generate)
graph.add_node("critique", critique)
graph.add_edge(START, "generate")
graph.add_edge("generate", "critique")
graph.add_conditional_edges("critique", should_continue, {
    "revise": "generate",  # Loop back!
    "done": END
})

Compare the first draft against the final version after three reflection cycles. The source emphasizes that the improvement is dramatic. This is one of the core patterns behind systems that appear to “think harder” on difficult problems.

03 · Smart Retrieval

Agentic RAG

Normal RAG: retrieve once, generate once. If the retrieved documents are poor, the answer is poor.

Agentic RAG: retrieve → evaluate quality → if poor, refine the query and retrieve again. This gives the system a recovery path when the first retrieval is weak.

The source contrasts Agentic RAG against naive RAG with a blunt summary: if retrieval misses key documents, the answer is wrong and there is no recovery.

Agentic RAGRetrieve → evaluate → refine → generate
class RAGState(TypedDict):
    query: str
    documents: list
    retrieval_count: int
    answer: str

def retrieve(state):
    docs = vector_store.similarity_search(state["query"], k=5)
    return {"documents": docs, "retrieval_count": state.get("retrieval_count", 0) + 1}

def evaluate_docs(state) -> str:
    eval_result = llm.invoke(
        f"Are these documents sufficient to answer: '{state['query']}'?\n"
        f"Documents: {[d.page_content[:200] for d in state['documents']]}\n"
        f"Answer YES or NO."
    )
    if "YES" in eval_result.content.upper() or state["retrieval_count"] >= 3:
        return "generate"
    return "refine"

def refine_query(state):
    better = llm.invoke(
        f"The search '{state['query']}' didn't find good results. "
        f"Suggest a better search query (just the query, nothing else)."
    )
    return {"query": better.content.strip()}

def generate(state):
    context = "\n".join([d.page_content for d in state["documents"]])
    answer = llm.invoke(
        f"Based on this context:\n{context}\n\nAnswer: {state['query']}"
    )
    return {"answer": answer.content}
04 · Parallelism

Map-Reduce — Parallel Processing

Map-reduce means processing multiple inputs in parallel and then merging the results into one report.

The source example is explicit: process 10 documents simultaneously, then merge the results into one report. It notes that this uses LangGraph’s Send API for fan-out parallelism.

05 · Structured Execution

Plan-and-Execute

Plan-and-execute separates planning from execution. A planner creates a multi-step plan, and an executor handles each step.

If a step fails, the planner can revise the remaining plan. The course compares this to a project manager creating a task list and a developer executing each task.

06 · Research Context

Key Research Papers

ReAct: Synergizing Reasoning and Acting

Yao et al., 2023 — ICLR

ReAct showed that interleaving reasoning with tool use beats either approach alone. The source notes a 6-point gain over chain-of-thought on HotpotQA, and explains that LangGraph’s agent loop directly implements this pattern.

Reflexion: Language Agents with Verbal Reinforcement Learning

Shinn et al., 2023 — NeurIPS

Reflexion introduced self-reflection with verbal self-feedback. The source highlights a jump to 97% success on AlfWorld versus 75% for ReAct, and maps LangGraph’s reflection loop to this idea.

Pregel: A System for Large-Scale Graph Processing

Malewicz et al., 2010 — Google, ACM SIGMOD

This is presented as the foundational architecture reference behind LangGraph. It introduced “think like a vertex,” where each vertex processes messages, updates state, and sends messages. The course also points out that Pregel’s checkpointing model influenced LangGraph persistence.
