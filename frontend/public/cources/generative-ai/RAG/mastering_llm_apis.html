Introduction to Mastering LLMs & APIs

Goal: Learn to talk to almost any AI model (GPT-4o, Claude, Gemini, Llama, DeepSeek, Grok, Mistral, Qwen and many more) using clean, smart code that lets you switch models with basically one line, without rewriting your entire app every time.

Why should we learn this?

In 2026 the LLM world changes every few months. Prices go up, new models drop, companies change rules, and suddenly your perfect GPT-4o code stops being the best choice.

Real situations you will face (and IΓÇÖve seen hundreds of developers hit this wall):

  • Your boss says ΓÇ£Switch to Claude, itΓÇÖs safer for our data and cheaper now.ΓÇ¥
  • OpenAI increases prices again ΓåÆ you want to jump to Groq + Llama or DeepSeek instantly.
  • You built something amazing with GPT-4o ΓåÆ now you want to test the same logic with Gemini 2.0 (huge context) or Grok for fun.
  • You need tools (search Google, check your shop database, send emails) but every model handles function calling differently.
  • You want the AI to remember your regular Bhopal customers (ΓÇ£Raman always wants fast deliveryΓÇ¥).

If your code is locked to only one company (like OpenAI), every change becomes painful and expensive. We learn to write code once ΓåÆ then switch models, add tools, memory, or fallback to cheaper options with almost zero effort.

Different types of LLMs youΓÇÖll meet in 2026

There are four main types of LLMs right now. Knowing them helps you choose the right one for the job:

  1. Proprietary / Closed-source models (Best quality, paid API)
    • OpenAI GPT-4o & o1 series: King of reasoning and complex tasks. Expensive but smartest overall. Great for coding, math, and careful thinking.
    • Anthropic Claude 3.5 Sonnet / Claude 4: Amazing at writing, coding, and following instructions safely. Very long context (200k tokens). Many people say it feels ΓÇ£more thoughtfulΓÇ¥.
    • Google Gemini 1.5 / Gemini 2.0: Insane context window (up to 1ΓÇô2 million tokens). Best when your document or chat history is huge. Also excellent with images and video.
  2. Open-weight / Open-source models (Free or cheap, run anywhere)
    • Meta Llama 3.1 / Llama 4 (70B or 405B): Extremely popular. Very strong reasoning. You can run them on Groq, Together.ai, or even locally.
    • DeepSeek-V3 / DeepSeek-R1: Crazy cheap and surprisingly smart, many people use it as daily driver now.
    • Mistral Large / Mixtral: European models, very good at multilingual (Hindi + English) and fast.
  3. Fast Inference Providers (Speed kings)
    • Groq: Super-fast chips. Llama or DeepSeek runs 5ΓÇô10├ù faster than normal. Perfect for chatbots and real-time replies.
    • Together.ai / Fireworks.ai: Great prices and fast. Good for running open models cheaply.
  4. Local / Private models (No API cost, runs on your laptop)
    • Use Ollama to run Llama 3.1, Mistral, or Phi-3 completely offline.
    • Perfect for sensitive data (medical, legal, or your shop customer info) because nothing leaves your computer.

Each type has different strengths, speed, cost, intelligence, safety, context length. The beauty of this course? YouΓÇÖll write code that can use any of them with one single change.

2. Core Idea: Unified LLM Caller (Write Once, Switch Forever)

LiteLLM makes 100+ models look like OpenAI format.

# llm.py ΓÇô your central file

from litellm import completion

import os

def ask_llm(

prompt_or_messages,

model="openai/gpt-4o-mini", # ΓåÉ change ONLY here!

temperature=0.7,

max_tokens=500,

tools=None, # for function calling later

response_format=None # for structured JSON

):

if isinstance(prompt_or_messages, str):

messages = [{"role": "user", "content": prompt_or_messages}]

else:

messages = prompt_or_messages

try:

response = completion(

model=model,

messages=messages,

temperature=temperature,

max_tokens=max_tokens,

tools=tools,

response_format=response_format,

api_key=os.getenv(f"{model.split('/')[0].upper()}_API_KEY") # auto picks key

)

return response.choices[0].message

except Exception as e:

print(f"Error with {model}: {e}")

return None

Test it:

def run_agent(query, model="groq/llama-3.1-70b-versatile"):

messages = [{"role": "user", "content": query}]

while True:

resp = ask_llm(messages, model=model, tools=tools)

messages.append(resp)

if not hasattr(resp, 'tool_calls') or not resp.tool_calls:

return resp.content # final answer

for tool_call in resp.tool_calls:

func_name = tool_call.function.name

args = json.loads(tool_call.function.arguments)

if func_name == "get_current_price":

# fake db call

7. Putting It All Together: Mini Shop Agent Example

Combine everything:

# shop_agent.py

def shop_agent(user_query):

# Retrieve long-term memory

docs = vectorstore.similarity_search(user_query, k=2)

memory_ctx = "\n".join(d.page_content for d in docs) or "No prior info."

system = f"You are helpful repair shop assistant in Bhopal. Use Hindi+English. Memory: {memory_ctx}"

messages = [{"role": "system", "content": system}, {"role": "user", "content": user_query}]

final = run_agent(messages, model="groq/llama-3.1-70b-versatile") # cheap & fast fallback

return final
