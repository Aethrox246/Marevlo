Reducing Hallucination in LLMs

Hallucinations happen when the model makes up facts or details that arenΓÇÖt true, like inventing a historical event. ItΓÇÖs a common issue because models predict based on patterns, not perfect knowledge. Some strategies to combat hallucinations include:

4.1 ReAct (Reason + Act) Prompting

ReAct is a prompting framework where an LLM combines reasoning steps and external actions (like searching or tool use), allowing it to gather information, update plans, etc.

How it works: A simple repeating loop of Thought ΓåÆ Action ΓåÆ Observation ΓåÆ repeat until the answer is ready.

Pros and Cons:

  • Pros: Enhances interpretability and accuracy in interactive or knowledge-intensive tasks, handles exceptions well.
  • Cons: Needs tool access (not plain chat); more complex to set up.

4.2 Chain-of-Verification (CoVe)

CoVe is a structured prompting method where the model first produces an answer, then deliberately verifies its own claims through targeted questions and revises the final response to reduce hallucinations and logical errors.

How it works:

The model generates an initial response, creates verification questions about its claims, answers them independently, and then revises the final answer using the verified facts.

Do you know? CoVe is like making the AI write an answer and then become its own strict teacher, it creates a checklist and fact-checks itself before finalizing!

Task: Summarize key features of a wireless earbud model accurately (no exaggeration).

CoVe Flow (multi-prompt):

  1. Draft Prompt: ΓÇ£Summarize the main features of XYZ wireless earbuds from the product specs: noise cancelling, 30-hour battery, IPX5 waterproof, deep bass.ΓÇ¥

  • (Questions: 1. What is the actual battery life claimed? 2. Is ΓÇ£top noise cancellingΓÇ¥ supported by specs? 3. What is the exact waterproof rating? 4. Does it say ΓÇ£best in classΓÇ¥?)
  1. Independent Answer Questions Prompt (one per question or batch): ΓÇ£Answer this verification question based only on original product specs: [question]ΓÇ¥
  • (Answers: 1. 30 hours. 2. No ΓÇö specs say ΓÇ£active noise cancellationΓÇ¥ but no ΓÇ£topΓÇ¥. 3. IPX5. 4. No such claim.)
  1. Revise Prompt: ΓÇ£Original draft: [draft]. Verification Q&A: [paste answers]. Revise the summary to be 100% factual and accurate.ΓÇ¥
  • (Final: ΓÇ£XYZ wireless earbuds feature active noise cancellation, 30-hour battery life, IPX5 water resistance, and deep bass sound.ΓÇ¥)

When to use it: Best for hallucination-prone tasks such as multi-hop question answering (e.g., HotpotQA), fact extraction, list-based queries, or long-form content generation where deliberate self-verification identifies inconsistencies better than single-pass methods like CoT, ensuring higher factual accuracy.

Pros/Cons:

  • Pros: Effectively reduces hallucinations and errors in complex, knowledge-intensive tasks by promoting self-correction; suitable for QA, verification, and generation needing coherence.
  • Cons: Requires multiple steps, increasing latency and compute; not ideal for simple, factual queries or real-time applications where speed is prioritized over verification.

4.3 Chain-of-Note (CoN)

Chain-of-Note prompting improves retrieval-augmented language models by forcing them to explicitly evaluate, filter, and summarize retrieved documents into structured ΓÇ£notes,ΓÇ¥ helping the model ignore noisy or irrelevant information and correctly admit when knowledge is insufficient.

How it works: The model first converts retrieved documents into concise relevance-checked notes, then reasons only over those validated notes to produce an answer or decide ΓÇ£unknown.ΓÇ¥

Do you know? When you give the AI lots of documents or search results, it often gets confused by junk. CoN forces it to read carefully, make short honest ΓÇ£notesΓÇ¥, and ignore the noise.Example 1: Answering Policy Question from Mixed Docs

Pros and Cons:

  • Pros: Enhances robustness in noisy environments by improving exact match accuracy and rejection rates for out-of-scope questions.
  • Cons: Increases response latency and token usage due to sequential note-taking, less efficient for clean or high-quality retrievals.

4.4 Chain-of-Knowledge (CoK)

This prompting method improves factual accuracy and complex reasoning by guiding the model to iteratively gather, verify, and integrate knowledge from multiple sources before producing an answer, reducing hallucinations compared to single-step reasoning.

How it works: The prompt forces the model to first frame the problem, then progressively retrieve and validate relevant knowledge from internal or external sources, and finally synthesize a grounded answer.

Do you know? This one makes the AI act like a detective ΓÇö it collects facts piece by piece from different areas, connects them, then gives a super grounded answer.

Prompt:

instruction: >

Build the knowledge chain step-by-step, then give a recommendation.

- Average daily sales (no discount)
- Expected sales lift from discount
- Profit margin (full vs discounted)

- Target customer segment
- Brand dilution risk

recommendation: "Yes | No"
suggested_discount: ""
reasoning: ""

input:
question: "<PUT_QUESTION_HERE>"

Input: Should we run a 25% flash sale on wireless earbuds (normal Γé╣2499, cost Γé╣1499) for the next 24 hours?

Chain steps:

  1. Current stock level and trend.
  2. Average daily sales without discount.
  3. Expected sales lift from 25% discount (use typical e-commerce accessory benchmarks).
  4. Profit margin at full price vs discounted price.
  5. Competitor pricing for similar earbuds.
  6. Customer segment most likely to buy (age, behavior).
  7. Risk of devaluing the brand or training customers to wait for sales.

Now connect the chain and give final recommendation: Yes/No + percentage + reasoning.

Output: Yes, but only 20% to protect margin.

MODULE 5

Context Optimization Techniques

Large Language Models (LLMs) now support very large context windows (often 128k to 1M+ tokens in modern models from 2025ΓÇô2026). However, real-world applications can still easily exceed these limits. Prompts may contain instructions, conversation history, examples, documents, and user inputs. When all of this information is included together, the total number of tokens can become too large.

Exceeding the token limit increases latency (response time), cost, and sometimes reduces model performance. Because of this, developers must manage how much information is included in the prompt.

Context optimization techniques help reduce the number of tokens while keeping the most important information for the task. The following sections explain common methods used to manage prompt context effectively in prompt engineering systems.

5.1 Naive Truncation

Naive truncation is the simplest and most commonly used method to ensure that a prompt fits within the modelΓÇÖs context window.

5.1.1 Removing extra tokens

In this approach, tokens are simply removed from the prompt when the total token count exceeds the allowed limit. Most LLM APIs use left truncation, which removes the oldest tokens first, such as earlier conversation messages or introductory content. This keeps the most recent user query and system instructions intact.

Some systems may use right truncation, which removes tokens from the end of the prompt. However, this is less common because recent information is usually more important.

Pros/Cons:

  • Pros: Zero additional compute, trivial to implement, predictable token usage.
  • Cons: Completely non-semanticΓÇöcritical information can be discarded arbitrarily. This often causes hallucinations, factual errors, or loss of user intent.

5.1.2 Sliding window approach

A slightly better approach is the sliding window method, which keeps the most recent context while summarizing or preserving key information from earlier parts.

For long documents or multi-turn conversations, the system can:

  • Extract relevant sections based on keywords or simple similarity checks
  • Maintain a fixed-size buffer of the most recent conversation turns
  • Optionally include a short summary of earlier interactions

Advanced retrieval systems use techniques such as:

  • Maximal Marginal Relevance (MMR): Retrieve documents that are both relevant and diverse, avoiding redundant information.
  • Contextual Compression Retrieval: Retrieve documents and compress them before sending them to the LLM.
  • Hierarchical Retrieval: Retrieve information in multiple levels instead of directly loading many chunks.

These approaches can reduce prompt size while improving answer accuracy by removing irrelevant information.

5.3 Natural Language Compression

More advanced techniques reduce prompt size by rewriting or summarizing content while keeping its meaning intact. These approaches can often compress text 5ΓÇô20├ù while maintaining useful information.

5.3.1 Prompt compression

Prompt compression techniques shorten prompts while attempting to maintain task performance.

Examples include methods such as LLMLingua and LongLLMLingua, which use smaller language models to identify and remove low-information tokens such as:

  • Stopwords
  • Redundant phrases
  • Repeated formatting or boilerplate text

Another category, called soft prompts, replaces long text instructions with learned token embeddings.

5.3.2 Token optimization

Token optimization focuses on reducing unnecessary tokens in the prompt.

Common strategies include:

  • Removing excessive formatting
  • Merging repeated instructions
  • Shortening descriptions without losing meaning

Some research methods estimate token importance using entropy or perplexity scores from smaller language models. This helps remove tokens that contribute little to the final output.

5.3.3 Semantic summarization

Semantic summarization uses an LLM to rewrite long content into a shorter, meaning-preserving summary that is relevant to the userΓÇÖs query.

One recent method is ACON (Agent Context Optimization) introduced in 2025. It is designed for long-running AI systems that accumulate large amounts of context.

ACON works through an iterative improvement process:

  1. Start with a simple summarization instruction (for example, ΓÇ£Summarize key facts, decisions, and unresolved tasksΓÇ¥).
  2. Identify cases where the compressed context causes errors.
  3. Use a stronger LLM to analyze why compression failed.
  4. Improve the summarization guideline.
  5. Repeat the process until performance stabilizes.

Using this method, systems can compress context by 30ΓÇô50% or more while maintaining about 95% of the original performance.

5.4 Dynamic Context Prompt Optimization (DCPO)

Dynamic Context Prompt Optimization adjusts the prompt during runtime as conversations or tasks progress.

5.4.1 Context rewriting

When a conversation becomes long (for example every 5ΓÇô10 turns or when the token usage approaches 80% of the limit), the system can ask the model to rewrite the context.

The rewritten context may:

  • Summarize previous conversation history
  • Extract important entities and decisions
  • Remove repeated or conflicting information
  • Rewrite instructions more concisely

This keeps the prompt manageable while preserving important details.

5.4.2 Prompt refinement loops

Use reflection or self-critique: After generating an output, the model evaluates whether the response meets the determined criteria. If improvements are needed, the system adjusts the prompt or context before generating the next output. Some frameworks also compress reasoning traces over time so that the prompt remains small while the task continues.

5.4.3 Adaptive prompts

Adaptive prompting dynamically changes how prompts are constructed depending on the situation.

Examples include:

  • Increasing or decreasing compression depending on the remaining token budget
  • Injecting stored memory when needed
  • Switching instruction style (detailed ΓåÆ concise) as the task becomes clearer

Some advanced agent frameworks also monitor token usage and automatically switch to summarized context or external storage when limits are approaching.
