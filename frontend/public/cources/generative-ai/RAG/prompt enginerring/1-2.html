1.1 Introduction to Prompt Engineering

1.1.1 What is Prompt Engineering

Prompt Engineering is the science of articulating your vision in a way that the large language models (LLMs) and vision-language models (VLMs) can understand and generate the desired output. It involves strategically designing task-specific instructions to guide the model output.

Think of it as a super-smart friend who can write stories, answer questions, or even design things, but he takes every single word you say literally. Say something vague like ΓÇ£make me something niceΓÇ¥ and you might get a random weird answer. But explain it step-by-step with all the details? Boom, perfect result every time!

1.1.2 Role of prompts in LLMs

Prompts are the primary way to control and improve LLM outputs. Without good ones, you just get boring, generic replies.

Do you know? Even big companies use prompt engineering instead of expensive model retraining. ItΓÇÖs like teaching the AI new tricks without touching its brain!

HereΓÇÖs a quick tip: the clearer your prompt, the better the answer. But donΓÇÖt go crazy long, too many details can confuse the AI and make it ΓÇ£hallucinateΓÇ¥ (make stuff up). Balance is the secret sauce.

1.1.4 Tokenization and context windows

LLMs donΓÇÖt read words like we do. They break text into pieces called tokens (about 3/4 of a word). They also have a memory limit called a context window.

Example:

  • Token: ΓÇ£Hello!ΓÇ¥ = 2 tokens ([ΓÇ£HelloΓÇ¥, ΓÇ£!ΓÇ¥]). ΓÇ£IΓÇÖm learning.ΓÇ¥ = 4 tokens ([ΓÇ£IΓÇ¥, ΓÇ£ΓÇÖΓÇ¥, ΓÇ£mΓÇ¥, " learningΓÇ£,ΓÇ¥."]).
  • Context Window: If a model has a 128K token window, itΓÇÖs like a notebook with 128,000 lines. Every prompt you give and reply it writes fills lines in this notebook. When itΓÇÖs full, it canΓÇÖt remember anything from the first page.

1.2 Context Engineering

Now letΓÇÖs level up! While prompt engineering is about writing good instructions, context engineering is about carefully designing and managing all the information the LLM sees in its ΓÇ£working memoryΓÇ¥ (the context window) so it can give better, more accurate, and consistent answers. It includes not just the prompt, but also extra details like past chats, important facts from documents, rules, tools, or user info.

1.2.1 Prompt vs Context

Think of it like this:

  • Prompt: The direct message you send to LLM.
  • Context: Everything (past chats, retrieved docs, etc.) send to the LLM along with the prompt.

1.2.2 Structuring LLM inputs

LLM inputs (what goes into the context window) work best when organized clearly, like sections in a book. Good structure helps the model find and use information easily, reduces confusion, and gives better results.

Common way to structure:

  • System instructions: Role and rules (e.g., ΓÇ£You are a helpful chef. Always be safe and list ingredients first.ΓÇ¥)
  • Examples: Show what good answers look like.
  • Retrieved or extra knowledge: Facts or documents needed for this task.
  • Conversation history: Past messages so it remembers the chat.
  • User question/input: What the user asks now.

1.2.3 Memory, instructions, and external knowledge

These are the three magic ingredients that fill the context window:

  • Instructions: Your job description for the AI
  • Memory: Short-term (this chat) + Long-term (user likes spicy food, hates onions)
  • External knowledge: Latest prices, documents, etc.

A meal-kit chatbot uses all three to never suggest onion recipes again.

1.3 Entity & Characterization

Entity & characterization is about giving the model a clear ΓÇ£identityΓÇ¥ and specific details so it behaves consistently and understands exactly who or what itΓÇÖs supposed to represent. This helps the chatbot feel like a real helpful person (or expert) instead of a boring robot. It makes responses more engaging, trustworthy, and tailored to the userΓÇÖs needs.

1.3.1 Attribute-based prompting

Instead of saying ΓÇ£find me a laptop,ΓÇ¥ you list exact must-haves:

"Recommend laptops with these attributes:

  • Price: under Γé╣60,000
  • Screen: 15-inch or larger
  • RAM: at least 16GB
  • Use: for college students (light gaming + studies)
  • Battery: 6+ hours Prioritize brands like Lenovo, HP, Dell. Include why each matches the attributes."

1.3.2 Persona prompting

Persona prompting means giving the model a full character or persona ΓÇö like a name, age, style, tone, background, and attitude. The AI ΓÇ£becomesΓÇ¥ that persona and answers as if itΓÇÖs a real person with that personality.

Example:

"You are Priya, a 28-year-old fashion-loving shopping assistant for a store in Mumbai.

  • Tone: super friendly, excited, like chatting with a best friend.
  • Style: trendy, honest ΓÇö tell if something doesnΓÇÖt suit.
  • Expertise: womenΓÇÖs ethnic wear, fusion outfits, budget tips.
  • Goal: help customers look amazing without overspending. ΓÇ¥

2.1 Zero-Shot Prompting

This is the simplest method where you give a direct instruction without any examples. Here, the model relies on prior knowledge to infer the task structure.

How it works: You have to state what you need clearly. To obtain high-quality zero-shot responses, the prompt should clearly define the task, provide necessary context, specify constraints, and indicate the desired output format. Optionally, assigning a role can help the model adopt domain-specific behavior.

Example 1:

instruction: Extract the "Fields" from the text

fields:
- Name
- Email
- Phone
- Skills

input_text: "<PASTE TEXT>"

Example 2: Generating product descriptions for an e-commerce store

Role: You are a professional e-commerce copywriter. Write a persuasive and engaging product description for an online store.

Context: Audience is online shoppers aged 18ΓÇô35 looking for trendy, value-for-money products

Product: Wireless neckband earphones

Description:
- Fast charging
- IPX5 water resistance
- 30-hour battery life
- Deep bass sound

Requirements:
- Highlight the most attractive features
- Include 3 clear customer benefits
- Add 1 short emotional or lifestyle hook (e.g., how it fits into daily life)
- Tone: friendly, confident, and exciting (but not exaggerated)
- Length: 140ΓÇô180 words

Output Format:
- Product Name
- One main paragraph
- One short bullet-point list of key features
- One closing call-to-action line

When to use it: For straightforward tasks where the topic is well known. ItΓÇÖs quick but might not handle complex questions perfectly.

Pros and Cons:

  • Pros: Fast and easyΓÇöno extra setup.
  • Cons: Can lead to errors for niche questions.

2.3 Prompt Templates

Prompt templates are ready-made reusable formats with placeholders (like [product_name]). You fill in the blanks for each use. They save time, keep consistency, and make it easy to scale.

2.3.1 Static templates

They have a fixed format with a few fill-in spots which is the user input or variable value. They are best for repetitive tasks with very similar outputs.

Example:

# Product Description Prompt

## Task
Write a short, exciting product description for **[product_name]**.

## Highlight
Focus on the following key features:
- **[key_feature1]**
- **[key_feature2]**

## Rules
- Keep the description under **100 words**.

## Call to Action
End the description with a clear call to action such as **"Order now!"**.

2.3.2 Dynamic templates

The template itself can change based on conditions (if-then logic). Different paths for different product categories or customer types.

Example:

# Customer Support Dynamic Prompt

## Persona
You are **[PERSONA]** for **[BRAND_NAME]**.

## Input
Customer Query: **[QUERY]**

## Logic

**If the query is about order status**
- Check the order tracking information.
- Provide the latest update.
- Respond politely and include a brief apology for any delay.

**If the query is about product recommendations**
- Suggest **2ΓÇô3 suitable products**.
- Include **product names and prices**.
- Keep the tone friendly and helpful.

**If the query is a complaint**
- Start with a **clear apology**.
- Acknowledge the issue.
- Offer a **solution or next step** (refund, replacement, or support).

## Response Guidelines
- Be polite and helpful.
- Keep responses clear and concise.
- Focus on solving the customer's problem quickly.

2.3.3 Parameterized prompts (for pros)

These are templates with many variables, usually built in code. You pass values programmatically.

Example

2.4 Prompt Evaluation
